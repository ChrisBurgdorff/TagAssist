Quality Control

  • Ensures dates are valid. User will get alert if date is earlier than 1980, later than the present day, or invalid number of months/days (for example, March 32).
  • Ensures dates make sense. User will get an alert if the Doc Date is later than the Deposit Date, etc.
  • Check Payor Type. If the user types a Payor that is on our Payor Types list, she will get an alert if she selected the incorrect Payor Type.
  • Check ABA / Routing Number. User will get an alert if the Endorsement Bank Account Number or the Payor Account Number is on our list of bank ABA / Routing numbers.
  • Ensure Deposit Date matches the Deposit Date from Deposit Slip.
  • Ensure Total Deposit Amount matches Total Deposit Amount from Deposit Slip.
  • Gives user alert if amount entered is greater than $150,000.
  • Spell Check. All fields that are subject to spelling errors are ran through our custom dictionary. The user is alerted to potential spelling errors.
